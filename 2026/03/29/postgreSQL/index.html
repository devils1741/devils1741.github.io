安装

安装之前需要先建立一个非root的用户，为了方便可以给管理员权限，因为pg设定上不允许以root的权限运行。

首先需要下载编译和安装，命令比较简单直接搞定。

# 安装依赖
sudo apt update
sudo apt install libreadline-dev flex bison zlib1g-dev perl
# clone code
git clone -b REL_15_2 https://gitee.com/mirrors/PostgreSQL.git --depth=1
# 编译安装
cd PostgreSQL
./configure --prefix=/home/test/pginstall --enable-debug
make -j6
make install

为了方便后续输入命令，将安装路径写入用户环境变量.

vim ~/.bashrc
# 文件最后一行添加
export PATH=/home/test/pginstall/bin:$PATH
# 刷新配置文件
source ~/.bashrc

然后可以简单的验证一下psql --version pg_ctl --version

PG的启动，停止和重启

PG在启动之前首先需要初始化

# 如果文件夹已经存在那么必须移除
rm -rf /home/postgres/pgdata
# 初始化
initdb -D /home/postgres/pgdata

初始化成功后会出现类似的提示:
Success. You can now start the database server using:
pg_ctl -D /home/postgres/pgdata -l logfile start

启动,并且指定日志的目录，便于后续出现问题时的debug

pg_ctl -D /home/postgres/pgdata -l logfile start

启动成功之后会有类似的输出:
waiting for server to start…. done
server started

启动之后可以使用命令查看pg是否真的在运行了

pg_ctl -D /home/postgres/pgdata status

如果需要停止可以使用命令:

pg_ctl -D /home/postgres/pgdata stop

也可以一步到位直接使用重启命令

pg_ctl -D /home/postgres/pgdata restart

PG内核的改动

要改数据库首先需要清楚PG各个功能模块在哪里。
最重要的模块是src/backend,几乎所有的数据库行为都在这里实现。
utils/adt – 增加内置函数
parser – SQL解析
optimizer – 执行计划
access/heap – 存储
access/transam – 事务
storage – lmgr
