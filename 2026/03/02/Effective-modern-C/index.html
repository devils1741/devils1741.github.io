类型推导

理解模板类型推导

这块内容不是很直观，有一些隐藏的规则，因此需要分场景进行解析。
假定代码为

template <typename T>
void f(ParamType param);
f(expr);

ParamType是一个指针或或着引用，但是不是万能引用

这种情况下如果expr的类型是一个引用，首先会忽略引用的部分，然后exper的类型会跟ParamType进行模式匹配决定T，
如果是指针的话，那么数据类型就会被严格推导为指针的数据类型。
为什么会这样设计？因为指针和引用的语义是不一样的，引用只是获取数据的方式，某些参数形式下推导阶段会忽略引用（如果保留引用，那么就成了 T && param，成了对引用的引用）
但是指针的*是数据类型的一部分所以会保留。

template <typename T>
void f(T& param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

int main() {
    int x = 27;
    const int cx = x;
    const int &rx = x;
    int &d = x;
    f(x);   // void f(T &) [T = int]
    f(rx);  // void f(T &) [T = const int]
    f(cx);  // void f(T &) [T = const int]
    f(d);   // void f(T &) [T = int]
}

ParamType是一个通用引用

分两种情况，如果传入一个右值，例如f(27)那么，T会被推导为int，也就是说右值T会推导为原本的数据类型
如果传入的是一个引用，那么就会触发引用折叠。
引用折叠的规则也很简单：只要里面有 &，结果就是 &

产生的类型 折叠结果
& & &
& && &
&& & &
&& && &&

为什么需要引用折叠？原因其实很简单，首先是C++不允许创建引用的引用，因为引用本身就是没有地址的。其次是
万能引用和引用折叠本身就是为了实现完美转发设计的，通过引用折叠能够确保推导完成之后还是能保持参数的性质

template <typename T>
void f(T &&param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

int main() {
    int x = 27;
    const int cx = x;
    const int &rx = x;
    f(x);  // void f(T &&) [T = int &]
    f(rx); // void f(T &&) [T = const int &]
    f(cx); // void f(T &&) [T = const int &]
    f(27); // void f(T &&) [T = int]
}

ParamType既不是指针也不是引用

如果ParamType不是指针或着引用，那么代表着函数是通过值传递的方式进行处理。
如果expr的类型是一个引用则忽略引用的部分，如果还是一个const或着volatile也忽略volatile。

template <typename T>
void f(T param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

int main() {
    int x = 27;
    const int cx = x;
    const int &rx = x;
    int &y = x;
    const char* const ptr = "Fun with pointers"; // 常量指针
    f(x);  // void f(T) [T = int]
    f(rx); // void f(T) [T = int]
    f(cx); // void f(T) [T = int]
    f(27); // void f(T) [T = int]
    f(y);  // void f(T) [T = int]
    f(ptr); // void f(T) [T = const char *]
}

参数退化

如果函数参数是数组，那么数组会退化成指针,但是通过引用可以避免。

template <typename T>
void f1(T param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

template <typename T>
void f2(T& param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

int main() {
    const char name[] = "J. P. Briggs";
    const char * ptrToName = name;
    int keyVals[] = { 1, 3, 7, 9, 11, 22, 35 };

    f1(name);    // void f1(T) [T = const char *]
    f1(ptrToName);   // void f1(T) [T = const char *]
    f2(name); // void f(T &) [T = const char[13]]
}

函数类型的参数也会退化为一个函数指针。

template<typename T>
void f1(T param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

template<typename T>
void f2(T & param) {
    std::cout << __PRETTY_FUNCTION__ << "\n";
}

void someFunc(int, double)
{}

int main() {
    f1(someFunc);  // void f1(T) [T = void (*)(int, double)]
    f2(someFunc);  // void f2(T &) [T = void (int, double)]
}
